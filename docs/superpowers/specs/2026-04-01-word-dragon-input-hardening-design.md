# Word Dragon v1 输入约束与内容字频修正设计

## 文档信息

- 日期：2026-04-01
- 项目：`word_dragon`
- 目标版本：`v1`
- 范围：关卡页信息收缩、错字拒绝输入、错误音反馈、候选池字频覆盖
- 状态：已确认，进入实施

## 背景

`M3R` 完成后，真机首关暴露出两类新问题：

- 关卡页仍保留 `释义提示` 卡片，占用宝贵竖屏空间，削弱了棋盘和字盘的主视觉。
- 首关 `level-0001` 的候选池字频不足，导致 `一国两制` 所需的 `两`、`一`、`制` 会被错误耗尽。
- 当前玩法允许把错字填入棋盘，这与常见成语填词游戏不一致，也会让老年用户误以为自己已经“填进去了”。

这些问题分别影响 UI 密度、内容正确性和玩法预期，必须在 `v1` 内收口。

## 目标

- 关卡页不再显示独立 `释义提示` 卡片。
- 候选字只有在与当前焦点格正确答案一致时才能落盘。
- 错字输入被拒绝时，提供即时本地错误音反馈。
- 内容生成与校验保证候选池频次足以覆盖整盘所有非共享格的用字需求。

## 非目标

- 不增加新的货币、评分或连击系统。
- 不引入联网资源或在线音效下载。
- 不在本轮重做章节页或首页。

## 方案结论

采用单一方案：

- 删除关卡页中的显式释义卡片，释义仅保留为当前线索上下文，供自动朗读与手动重播使用。
- 领域状态机改为“只接受正确字”：
  - 正确字写入当前焦点格并推进焦点。
  - 错误字不写入棋盘，不改变候选池剩余次数。
  - 通过一个本地短错误音提示用户本次输入被拒绝。
- 内容生成器输出候选池时，按棋盘真实需求频次生成，而不是简单去重。
- 内容校验器新增“候选池频次覆盖”检查，防止错误资产再次落盘。

## 关键规则

### 1. 释义上下文

- 关卡页不再展示独立文本卡片。
- 当前选中成语的释义仍保留在状态机中，用于：
  - 自动朗读
  - 手动点击 `重播发音`
  - 提示功能的内部上下文

### 2. 错字拒绝

- 每次候选字点击都与当前焦点格的正确答案比较。
- 若不匹配：
  - 棋盘不变
  - 焦点不变
  - 候选池剩余次数不变
  - 播放错误音
- 若匹配：
  - 按既有焦点推进规则继续
  - 候选池按正确占用次数即时刷新

### 3. 候选池频次

- 候选池统计基于棋盘真实占用需求，而不是“本关出现过哪些字”。
- 对重复字：
  - 非共享格上每出现一次，就必须在候选池中有一次可用次数。
  - 共享格只按一个格子计算一次。

## 验收

- 页面中不存在 `释义提示` 卡片文案。
- 错误候选字点击后，目标格保持空白或原正确字，不允许出现错填状态。
- 错误候选字点击会触发一次本地错误音。
- 内容校验脚本能检出候选池频次不足的关卡。
- 重生成后的 `level-0001` 候选池能支持首关完整填完，不再出现 `两` 被过早耗尽。
