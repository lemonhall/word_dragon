# ECN-0001: Word Dragon 关卡交互与 TTS 玩法重构

## 基本信息

- **ECN 编号**：ECN-0001
- **关联 PRD**：PRD-0001
- **关联 Req ID**：`REQ-0001-004`、`REQ-0001-005`、`REQ-0001-006`、`REQ-0001-007`
- **发现阶段**：`v1` 手工试玩回顾后，进入玩法重构前
- **日期**：2026-04-01

## 变更原因

真机试玩表明，当前 `v1` 关卡页虽然能跑通候选字盘、提示、TTS 和存档恢复，但玩法交互偏离常见成语填词游戏的直觉：

- 页面把注意力从棋盘分散到 `当前成语` 和 `成语列表` 两块面板。
- 用户不是通过点击棋盘直接进入输入态，而是走“先选成语，再填字”的旁路。
- 候选字没有剩余次数和置灰反馈，用户无法直观判断哪些字还可用。
- TTS 直接朗读答案成语文本，破坏猜题过程。

这些问题会直接损害核心玩法体验，因此需要在 `v1` 内完成设计修正，而不是延后到后续版本。

## 变更内容

### 原设计

- `REQ-0001-004`：候选字按钮输入当前选中空格，但没有规定棋盘必须作为主交互入口。
- `REQ-0001-005`：点选成语显示简短释义，默认通过显式释义区承载。
- `REQ-0001-006`：点选成语时自动朗读“成语 + 简短释义”。
- `REQ-0001-007`：恢复未完成盘面，但未要求恢复选中成语和焦点格。

### 新设计

- 关卡页改为**纯棋盘驱动**：
  - 删除 `当前成语` 和 `成语列表` 两块显式面板。
  - 用户通过点击棋盘格选择当前成语和当前焦点格。
- 候选字盘改为**剩余次数驱动**：
  - 按“关卡原始候选池 - 当前棋盘占用”计算剩余次数。
  - 剩余次数为 `0` 时置灰并禁用。
- TTS 改为**线索/反馈驱动**：
  - 不再朗读成语答案文本。
  - 仅朗读释义提示、操作反馈和通关反馈。
- 本地恢复改为**状态恢复**：
  - 除盘面外，还需恢复当前选中成语、当前焦点格和完成态。
