## Vision

交付一个完全离线、面向老年用户的 Android 中文四字成语填词 App。用户打开 App 后可以直接继续上次进度，也可以进入章节选关；在横屏或竖屏下都能使用大字号完成纵横交叉的成语填字；用户通过点击棋盘进入输入态，并可通过 Android 系统 TTS 听到释义提示和状态反馈。`v1` 必须内置上千关固定关卡和常用成语数据包，且所有关键流程都能在无网络环境下稳定完成。

## 用户与场景

### 主要用户

- 50 岁以上、喜欢中文文字游戏的安卓用户
- 不愿意注册登录、不愿意联网折腾、偏好直接开始玩的用户
- 需要更大字体、更大点击区域、更少操作层级的用户

### 核心使用场景

- 用户在地铁、家里或无网络环境中打开 App，点击“继续游戏”直接恢复到上次关卡。
- 用户在章节页选择一关，通过点击棋盘格切换当前成语与输入焦点，并用候选字完成填词。
- 用户切换当前成语时可听到释义提示，在卡住时用提示继续推进。

## 成功标准

- Android 设备在断网条件下可从冷启动进入首页并打开关卡。
- 内置关卡数不少于 `1000` 关。
- 任何单关只使用四字成语，且每关成语数在 `4-8` 之间。
- 首页“继续游戏”能恢复上次未完成盘面。
- 关卡在横屏和竖屏下都满足最小字号和最小触控尺寸要求。
- TTS 初始化失败不会阻断关卡游玩。

## 非目标

- 登录、排行榜、云存档、社交分享
- 动态出题、每日挑战、联网更新词库
- 体力、计时、商城、广告、货币系统
- 四字成语以外的诗词、歇后语、俗语玩法

## 需求清单

### REQ-0001-001 完全离线单机运行

- **动机**：目标用户不应被网络、账号或服务端依赖阻断。
- **范围**：App 的首页、章节、关卡、提示、TTS 开关、存档恢复均可在无网络条件下使用。
- **非目标**：不要求离线语音合成引擎随 App 分发；系统 TTS 是否安装由设备决定。
- **优先级/阶段**：P0 / v1
- **验收口径**：
  - 飞行模式下安装并启动 App，首页可见 `继续游戏`、`章节选关`、`设置` 三个入口。
  - 飞行模式下可进入任意已打包关卡并完成填字。
  - 断网不触发必须登录、必须联网或必须更新的阻断弹窗。

### REQ-0001-002 仅使用常用四字成语

- **动机**：老年用户需要常用、可理解、不过分生僻的内容。
- **范围**：内置可玩成语必须是四字成语，并经过常用度筛选。
- **非目标**：不追求成语全收录，也不纳入低频冷僻条目凑数量。
- **优先级/阶段**：P0 / v1
- **验收口径**：
  - 成语数据包中每条 `text` 长度为 `4` 个汉字。
  - 内容校验脚本报告 `enabled=true` 的词条全部通过四字规则。
  - 内容校验脚本可输出常用度分层统计。

### REQ-0001-003 预生成固定关卡，单关 4 到 8 个成语

- **动机**：固定关卡比运行时出题更稳定，便于控制可玩性和 UI 质量。
- **范围**：所有关卡在开发期预生成并打包，运行时只读取关卡文件。
- **非目标**：不在客户端动态拼盘，也不生成无限题库。
- **优先级/阶段**：P0 / v1
- **验收口径**：
  - 每关成语数满足 `4 <= idiom_count <= 8`。
  - 每关字盘连通，无孤立词条。
  - 客户端运行时不包含动态关卡生成逻辑入口。

### REQ-0001-004 候选字盘填词交互

- **动机**：老年用户更容易理解候选字盘，而不是系统输入法。
- **范围**：关卡页通过点击棋盘格选择当前成语和输入焦点，并通过底部大号候选字按钮输入当前焦点格，不弹系统键盘。[已由 ECN-0001 变更]
- **非目标**：不提供拼音输入、手写输入或全键盘模式。
- **优先级/阶段**：P0 / v1
- **验收口径**：
  - 点击棋盘格后，当前输入焦点会落到被选中的合法格子上。
  - 点击候选字按钮后，当前焦点格内容立即更新。
  - 玩家可修正错误填字，不需重开关卡。
  - 关卡页不会主动弹出系统输入法。

### REQ-0001-005 释义提示与提示机制

- **动机**：用户需要通过简短释义理解成语，卡关时也需要温和的辅助。
- **范围**：点选棋盘上的成语后更新当前线索上下文，并允许通过发音按钮获得对应释义提示；提供“提示一个字”和“揭示整条成语”。[已由 ECN-0001 变更]
- **非目标**：不提供长篇词典页，不提供付费提示体系。
- **优先级/阶段**：P0 / v1
- **验收口径**：
  - 点选成语时，当前线索上下文更新为对应释义提示。
  - “提示一个字”只揭示当前成语中一个未正确填写的字。
  - “揭示整条成语”可直接完成当前成语。

### REQ-0001-006 Android TTS 朗读

- **动机**：语音朗读能降低识字负担并增强上手体验。
- **范围**：点选成语时自动朗读释义提示和状态反馈，并支持手动重播与关闭；TTS 不得朗读成语答案文本。[已由 ECN-0001 变更]
- **非目标**：不做第三方云 TTS，不要求自定义音色下载。
- **优先级/阶段**：P1 / v1
- **验收口径**：
  - 在系统 TTS 可用时，点选成语会触发一次朗读。
  - 自动或手动朗读内容不包含成语答案文本。
  - 用户可关闭自动朗读，关闭后仍可手动重播。
  - TTS 初始化失败时，关卡仍能正常游玩并提示失败原因。

### REQ-0001-007 本地进度保存、继续游戏与章节选关

- **动机**：老年用户需要简单、稳定的恢复路径，也需要能回头重玩。
- **范围**：本地自动保存当前关卡状态、解锁进度、提示使用情况、当前选中成语与当前焦点格；首页提供继续游戏与章节选关。[已由 ECN-0001 变更]
- **非目标**：不做云同步，不做多账号切换。
- **优先级/阶段**：P0 / v1
- **验收口径**：
  - 填字中途退出并重新打开 App 后，可恢复到上次未完成盘面。
  - 恢复未完成盘面时，当前选中成语和焦点格也能恢复。
  - 已通关关卡可在章节页重玩。
  - 通关后下一关自动解锁。

### REQ-0001-008 老年友好视觉与横竖屏适配

- **动机**：大字号、清晰高亮和大触控区域是核心用户体验，不是加分项。
- **范围**：横屏与竖屏两套布局都满足最小字号与最小触控尺寸，并保留清晰高亮。
- **非目标**：不做桌面端、平板特化布局作为 v1 必交项。
- **优先级/阶段**：P0 / v1
- **验收口径**：
  - 关卡布局校验脚本对横屏和竖屏均返回通过。
  - 高亮状态不只靠颜色表达，至少同时包含边框或底色差异。
  - 候选字按钮和主要操作按钮满足最小触控尺寸阈值。

### REQ-0001-009 内置上千关内容包

- **动机**：产品需要具备长期可玩性，不能只有几十关演示内容。
- **范围**：App 首版打包的章节索引与关卡总数不少于 `1000`。
- **非目标**：不要求首版做到无限题库。
- **优先级/阶段**：P0 / v1
- **验收口径**：
  - `chapters.json` 或等价索引中统计出的关卡总数 `>= 1000`。
  - 内容校验脚本验证所有关卡文件均能被索引引用。
  - 任一章节均不存在缺失关卡文件。

### REQ-0001-010 内容来源留痕与校验

- **动机**：词库来源、常用度筛选和关卡生成规则必须可追溯，避免后续数据漂移。
- **范围**：仓库中保留内容来源说明、生成脚本和校验脚本。
- **非目标**：不要求在 App 内展示全部词库来源说明页面。
- **优先级/阶段**：P1 / v1
- **验收口径**：
  - 内容构建与校验命令可在本地重复运行并返回确定结果。
  - 词库来源和筛选策略在仓库文档中有明确记录。
  - 校验脚本能检测四字规则、关卡连通性、重复关卡和索引断链。

### REQ-0001-011 核心流程自动化验证

- **动机**：`v1` 范围不小，没有自动化验证会导致回归成本失控。
- **范围**：对核心领域逻辑和至少一条完整用户流程建立自动化测试。
- **非目标**：不要求 `v1` 达到全量 UI 截图测试覆盖。
- **优先级/阶段**：P1 / v1
- **验收口径**：
  - 单元测试覆盖关卡状态机、提示逻辑、存档恢复和 TTS 文案组装。
  - 仪器测试覆盖从首页进入关卡并完成一次通关的流程。
  - 屏幕旋转后的状态恢复流程有自动化验证。

## 术语

- **成语词条**：一条可玩四字成语及其附属数据。
- **关卡**：由 `4-8` 个成语构成的固定字盘。
- **章节**：关卡分组及其索引。
- **候选字盘**：用于填字的大号汉字按钮区。
- **进度快照**：当前未完成关卡的盘面状态与提示使用记录。
